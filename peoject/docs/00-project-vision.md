# 项目愿景

## 项目名称

FlowDesk

## 一句话介绍

FlowDesk 是一个基于 Spring Boot 的企业级工单协同平台，用于展示面向技术面试的生产级后端开发能力。

## 目标场景

在企业内部，员工会针对 IT、运维、人事或业务问题提交支持工单。系统负责将工单路由到合适团队，跟踪处理进度，支持评论和附件，发送通知，记录审计日志，并提供运营统计数据。

## 面试定位

这个项目应该帮助开发者讲清楚：

- 一个后端请求如何从 Controller 流转到数据库
- 认证和授权如何处理
- 数据权限如何落地
- 如何建模业务流程
- 如何设计事务和最终一致性
- 如何让异步通知可靠
- 如何处理文件、日志、统计和搜索
- 如何组织企业级项目模块

## 业务模块

核心模块：

- 认证
- 用户和组织管理
- RBAC
- 工单流程
- 评论协作
- 附件
- 通知
- 操作时间线
- 报表和统计

高级模块：

- 多租户
- 状态机
- Outbox 事件
- MQ 重试和幂等
- Redis 缓存
- 全文搜索
- 限流
- 可观测性

## 非目标

FlowDesk 初期不应该变成一个过大的系统。

避免：

- 过早构建完整低代码工作流引擎
- 在后端基础不稳定前投入复杂前端 UI
- 在模块化单体尚未清晰前引入微服务
- 没有明确业务价值就引入大量中间件

## 成功标准

项目达到以下状态即可认为成功：

- 可以按照清晰说明在本地启动
- 可以演示完整工单核心流程
- 重要后端设计决策有文档记录
- API 结构清晰且可测试
- 开发者能用面试语言讲清楚每个主要功能
