# ADR-004: 使用 Outbox 保证事件一致性

## 状态

Proposed

## 背景

当工单创建或更新时，系统可能需要通过 MQ 发送通知。如果数据库事务成功，但 MQ 发布失败，通知事件可能丢失。

## 决策

使用 Outbox pattern。

业务数据和事件数据在同一个本地数据库事务中保存。独立 publisher 负责发送待发布事件到 MQ，并标记为已发布。失败事件可以重试。

## 影响

收益：

- 避免数据库和 MQ 之间的事件丢失
- 支持重试和可观测性
- 明确表达最终一致性

代价：

- 增加 `outbox_event` 表
- 需要 publisher 和重试逻辑
- 消费者仍然必须幂等

## 备选方案

- 在事务中直接发送 MQ：实现简单，但 MQ 失败时不可靠。
- 分布式事务：过重，本项目没有必要。
