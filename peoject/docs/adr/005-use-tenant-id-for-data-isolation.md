# ADR-005: 使用 Tenant ID 做数据隔离

## 状态

Proposed

## 背景

FlowDesk 按 SaaS 化企业系统设计。不同租户不能访问彼此的工单、文件、用户和日志。

## 决策

租户范围内的业务表使用 `tenant_id`，并在业务查询中应用租户过滤。

租户上下文来自已认证用户。

## 影响

收益：

- 简单且常见的 SaaS 隔离模型
- 查询和运维相对方便
- 适合单数据库部署

代价：

- 每个查询都必须遵守租户范围
- 遗漏租户过滤可能导致数据泄露
- 平台管理员操作需要谨慎设计

## 备选方案

- 每个租户独立数据库：隔离更强，但运维成本高。
- 每个租户独立 schema：隔离程度适中，但迁移复杂度更高。
