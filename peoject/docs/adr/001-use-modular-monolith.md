# ADR-001: 使用模块化单体

## 状态

Accepted

## 背景

FlowDesk 是一个面向技术面试的后端项目。它需要展示企业级架构和真实后端设计，但也必须保持本地易运行、易测试、易讲解。

如果一开始就采用微服务，会引入服务发现、分布式链路追踪、部署编排、分布式事务等额外复杂度。

## 决策

初始架构采用模块化单体。

各模块需要保持清晰边界，但运行在同一个 Spring Boot 应用中。

## 影响

收益：

- 本地开发更简单
- 调试和测试更方便
- 部署复杂度更低
- 仍然可以保持清晰模块边界
- 后续必要时可以演进为微服务

代价：

- 需要团队或个人自律，避免跨模块随意耦合
- 模块边界更多依赖架构约定，而不是物理部署边界

## 备选方案

- 微服务：第一版基础设施成本过高。
- 简单单包 CRUD 应用：无法充分展示面试所需的后端能力。
