# ADR-003: 工单流程使用状态机

## 状态

Proposed

## 背景

工单状态变更包含业务规则。例如，已关闭工单不能再次处理，待确认工单可以关闭，也可以退回处理中。

如果流转逻辑散落在多个 Service 中，会难以维护，也容易产生规则不一致。

## 决策

将工单状态流转规则集中到类似状态机的组件中。

该组件负责校验：

- 源状态
- 目标状态
- 操作人角色
- 必填字段
- 时间线和事件等副作用

## 影响

收益：

- 避免状态判断散落在 if-else 中
- 工作流规则更容易解释
- 状态流转测试更容易编写

代价：

- 需要提前建模状态和事件
- 如果第一版直接做完整工作流引擎，可能过度设计

## 备选方案

- 在 Service 方法中直接写 if-else：初期简单，但后期难维护。
- 引入完整工作流引擎：能力强，但第一版过重。
